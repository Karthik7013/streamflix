# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately by opening a GitHub Security Advisory or emailing the project maintainers.

Please do **not** report security vulnerabilities through public GitHub issues.

We will acknowledge receipt within 48 hours and provide a timeline for a fix.

## Scope

This security policy covers:
- Authentication and session management
- Data privacy and access control
- API endpoint security
- Dependency vulnerabilities

Third-party services (Better Auth, Vercel, Neon, Upstash, Internet Archive, TMDB) have their own security policies and are outside the scope of this document.
